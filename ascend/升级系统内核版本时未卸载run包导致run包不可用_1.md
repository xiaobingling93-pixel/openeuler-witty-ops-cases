# 升级系统内核版本时未卸载run包导致run包不可用

## 内核版本


## 问题现象
在安装了run包的环境中直接升级内核，重启后系统启动异常，且后续即使重新安装run包也无法使用。

## 问题根因
安装run包时驱动ko模块被加载到内核中，升级内核版本会触发根文件系统打包过程，将原环境中的旧ko模块打包进新的根文件系统，导致系统启动异常和run包不可用。

## 解决方案
1. 手动卸载相关ko模块：先通过'lsmod|grep drv'查询当前系统中与驱动相关的ko模块，然后使用'rmmod ko1 ko2 ...'命令依次卸载所有相关ko模块；2. 重新打包根文件系统：执行'dracut --force'命令；3. 重启系统：执行'reboot'命令。
