# 启用进程级在线恢复后，报错There is unsafe data in the input tensor，恢复失败

## 内核版本


## 问题现象
训练任务在发生故障后启用进程级在线恢复（Step级别重计算恢复）功能，在恢复完成后继续训练，但在恢复训练后的第一个迭代尚未完成时，再次报错“There is unsafe data in the input tensor”，导致恢复失败。

## 问题根因
故障发生时，涉及该故障的Tensor会被标记为unsafe data，数据不再可信。如果该Tensor是全局变量且未被正确重建或修复，在后续训练中被访问时会触发错误。特别是当训练框架中存在未被故障恢复机制跟踪的全局Tensor时，容易导致此类问题。

## 解决方案
根据Tensor与训练迭代的依赖关系采取不同措施：1）若Tensor与训练迭代无依赖关系（如仅用于记录日志的全局变量），应在rollback阶段重新初始化或释放；2）若Tensor与训练迭代有依赖关系且与副本优化器映射一致，则需在repair阶段通过点对点通信重建并修复其数据。同时，应避免在重建前访问被标记为unsafe的Tensor，并排查所有全局Tensor确保其被故障恢复框架正确跟踪和处理。
