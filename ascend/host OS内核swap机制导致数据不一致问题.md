# host OS内核swap机制导致数据不一致问题

## 内核版本
6.6以下

## 问题现象
用户通过acl接口进行host侧内存（malloc申请的内存）和device侧内存拷贝时，概率性发生拷贝后数据不一致的问题。具体表现为：先将host内存va0拷贝到device侧，再将device内存拷贝回host侧内存va1，发现va0与va1内容不一致。

## 问题根因
OS内核（版本6.6以下）的swap机制在内存紧张时会将不常用的内存页换出到磁盘。若在aclrtMemcpy拷贝操作前，相关host内存已被swap换出，则实际拷贝的数据为非预期内容，从而导致数据不一致。

## 解决方案
升级OS内核版本至6.6或更高版本，该版本已修复此swap机制引发的数据不一致问题。
