# 服务器异常复位或下电导致重新安装驱动或固件失败

## 内核版本


## 问题现象
在Atlas 200I SoC A1核心板、Atlas 300I Pro推理卡、Atlas 300V Pro视频解析卡、Atlas 300I Duo推理卡、Atlas 300V视频解析卡的驱动或固件安装或升级过程中，若服务器发生异常复位或下电，重启后重新安装驱动时会报错“Device_images_crl_check failed”，重新安装固件时会报错“ERR_NO:0x0080;ERR_DES:install path(Binary file /etc/ascend_install.info matches) is wrong, install failed”。

## 问题根因
服务器异常复位或下电导致驱动或固件安装过程被中断，系统中残留了不完整的安装状态或校验文件（如/root/ascend_check），使得后续重新安装时校验失败或路径判断错误。

## 解决方案
对于驱动安装报错，需先删除“/root/ascend_check”文件，然后卸载驱动并重新安装；对于固件安装报错，需先卸载固件，再重新安装。具体操作步骤参考《NPU驱动和固件安装指南》中的相关章节。
