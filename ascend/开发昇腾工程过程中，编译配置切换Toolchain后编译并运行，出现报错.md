# 开发昇腾工程过程中，编译配置切换Toolchain后编译并运行，出现报错

## 内核版本


## 问题现象
在开发昇腾工程过程中，改变编译配置中的Toolchain，切换了编译架构，再次编译并在对应架构的环境中运行时，出现“cannot execute binary file: Exec format error”错误。

## 问题根因
工程内容未做任何改动的情况下，切换编译配置改变编译架构，并使用了“Build Project”增量编译，导致最终编译结果未重新生成，仍为旧架构的二进制文件，因此在新架构环境中无法执行。

## 解决方案
工程切换编译配置后，应使用“Rebuild Project”进行全量编译，或先使用“Clean Project”清理之前的编译结果，再使用“Build Project”重新编译，以确保生成与目标架构匹配的可执行文件。
