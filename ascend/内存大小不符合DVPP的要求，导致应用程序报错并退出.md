# 内存大小不符合DVPP的要求，导致应用程序报错并退出

## 内核版本


## 问题现象
应用程序运行时因DVPP内存大小不足而报错退出，日志中出现类似“buffer size(3110400) is smaller than need buffer size(4147200) when format is 3”或“vpc end address is illegal, check allocated buffer size”的错误信息。

## 问题根因
虽然使用了正确的DVPP内存申请接口，但申请的内存大小小于该格式所需的输入或输出内存大小；或者传入接口的内存大小虽与格式匹配，但超出了实际申请的内存大小，导致校验失败、结束地址非法。

## 解决方案
1. 参考昇腾文档中心推理部分的API文档，在媒体数据处理章节中根据DVPP各功能对内存大小的要求，检查代码中对应格式的内存大小是否正确；2. 在代码中增加打印内存长度的日志，确保传入接口的内存大小与实际申请的内存大小一致。
