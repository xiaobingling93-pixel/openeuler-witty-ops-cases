# 在模型运行时遇到报错“Inner Error xxxx”，但打印的堆栈报错信息与实际错误无关

## 内核版本


## 问题现象
在NPU上进行模型训练时，出现“Inner Error xxxx”报错，但打印的堆栈信息与实际错误不一致，难以定位真实问题。

## 问题根因
NPU模型训练默认采用异步运行模式，导致错误发生时打印的堆栈信息并非实际出错位置。

## 解决方案
通过设置环境变量 ASCEND_LAUNCH_BLOCKING=1，将运行模式改为同步模式，从而获得与实际错误一致的堆栈信息。具体操作为：export ASCEND_LAUNCH_BLOCKING=1，然后重新运行模型。
