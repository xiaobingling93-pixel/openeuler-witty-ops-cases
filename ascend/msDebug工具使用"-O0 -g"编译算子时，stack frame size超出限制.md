# msDebug工具使用"-O0 -g"编译算子时，stack frame size超出限制

## 内核版本


## 问题现象
使用msDebug工具以"-O0 -g"选项编译自定义算子时，编译报错提示栈帧大小（16024字节）超过硬件限制（16000字节），导致编译失败。

## 问题根因
核函数中使用的栈空间过大，超过了昇腾硬件对单个函数栈帧大小的限制。这通常是因为Ascend C相关函数被强制内联（inline），导致函数调用展开后局部变量和临时对象占用过多栈空间。

## 解决方案
有两种解决方法：1）手动添加编译选项 --cce-ignore-always-inline=true，以忽略Ascend C函数的always_inline属性，使函数正常跳转而非内联，从而减少栈空间占用；2）升级编译器至2024年7月或之后的版本，该版本在使用-O0 -g选项时会自动启用--cce-ignore-always-inline=true选项。
