# 虚拟机安全启动打开导致驱动安装失败

## 内核版本


## 问题现象
在Atlas 200T A2 Box16异构子框、Atlas 800T A2训练服务器、Atlas 800I A2推理服务器、Atlas 900 A2 PoD集群基础单元的NPU芯片直通到虚拟机的场景下，虚拟机中安装驱动包时出现“[ERROR]Driver_ko_install failed, details in : /var/log/ascend_seclog/ascend_install.log.”报错。通过mokutil --sb-state命令查询显示“SecureBoot enabled”，表明安全启动已开启。

## 问题根因
虚拟机安全启动开启时，操作系统会拒绝加载未签名的内核模块（ko文件），导致驱动安装过程中插入驱动模块失败。

## 解决方案
关闭虚拟机的安全启动：1. 重启虚拟机并在宿主机使用virsh console <vm name>连接，快速按F5进入BIOS；2. 进入Device Manager → Secure Boot Configuration；3. 选择Attempt Secure Boot并确认；4. 选择Reset Secure Boot Keys并确认为Yes；5. 退出并选择Reset使设置生效；6. 重新登录虚拟机并按照官方指南重新安装驱动。
