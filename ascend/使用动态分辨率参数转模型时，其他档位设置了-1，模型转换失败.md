# 使用动态分辨率参数转模型时，其他档位设置了-1，模型转换失败

## 内核版本


## 问题现象
使用ATC工具进行模型转换时，使用--dynamic_image_size参数转换支持多个分辨率的模型，提示E10019报错信息：'When [--dynamic_image_size] is included, only the height and width axes can be –1 in [--input_shape].'

## 问题根因
使用--dynamic_image_size参数时，输入shape中只有高度（H）和宽度（W）维度允许设置为-1，且仅支持NCHW或NHWC格式。原命令中将batch（N）、H、W均设为-1，违反了该限制。

## 解决方案
修改--input_shape参数，仅将H和W设为-1，例如：--input_shape="Placeholder:1,-1,-1,3"，确保除H、W外的其他维度不为-1。
