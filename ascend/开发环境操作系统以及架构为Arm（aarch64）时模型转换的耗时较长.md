# 开发环境操作系统以及架构为Arm（aarch64）时模型转换的耗时较长

## 内核版本


## 问题现象
在开发环境的操作系统和架构为Arm（aarch64）时，执行模型转换操作耗时较长。

## 问题根因
架构内核未指定最优编号的CPU内核，导致模型转换性能不佳。

## 解决方案
1. 以ATC安装用户登录开发环境，执行su root命令切换到root用户；2. 确保开发环境已连接网络后，执行yum -y install numactl命令安装numactl工具；3. 切换回ATC安装用户，使用numactl -C 16-31 --localalloc <args>命令指定编号16到31的CPU核来处理模型转换操作，以提升性能（用户可根据实际情况调整CPU核编号范围）。
