# 量化感知训练场景下，出现数据类型不一致的解决方法

## 内核版本


## 问题现象
在混合精度训练场景下进行量化感知训练时，调用create_quant_retrain_model接口后，脚本报错：For 'Conv2D', type of `w` should be same as `x`, but `x` with type Tensor[Float16] and `w` with type Ref[Tensor(F32)]，即输入张量与权重张量的数据类型不一致。

## 问题根因
调用create_quant_retrain_model接口后，原始网络中的nn.Dense和nn.Conv2D被分别替换为DenseQatBlock和Conv2dQatBlock对象，但用户在切换精度的代码中未将这些新的量化感知训练模块纳入类型判断范围，导致这些模块的权重未被正确转换为float32，从而与输入的Float16类型不匹配。

## 解决方案
修改量化感知训练脚本，在switch_precision函数中导入DenseQatBlock和Conv2dQatBlock，并在类型判断时同时检查原始模块和对应的QAT模块。具体为：from amct_mindspore.cells.dense_qat_cell import DenseQatBlock；from amct_mindspore.cells.conv2d_qat_cell import Conv2dQatBlock；并将isinstance(cell, (nn.Dense, ))扩展为isinstance(cell, (nn.Dense, DenseQatBlock))，对Conv2D做类似处理。
