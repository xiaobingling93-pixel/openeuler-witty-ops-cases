# 多张Atlas 中心推理卡直通虚拟机场景下，虚拟机中所有命令执行缓慢

## 内核版本


## 问题现象
在多张Atlas中心推理卡直通虚拟机的场景下，虚拟机中所有命令执行缓慢，包括驱动npu-smi命令或dcmi接口响应缓慢、卡顿。

## 问题根因
创建虚拟机时分配的内存空间或CPU核数不足。

## 解决方案
1. 关闭虚拟机：virsh shutdown <虚拟机名称>；2. 登录物理机；3. 查询虚拟机名称：virsh list --all；4. 查询剩余内存：free -h；5. 编辑虚拟机配置文件：virsh edit <虚拟机名称>，根据业务需求增加内存（如64GiB）和vCPU数量（如8核）；6. 保存配置（wq!）；7. 启动虚拟机使配置生效：virsh start <虚拟机名称>。
