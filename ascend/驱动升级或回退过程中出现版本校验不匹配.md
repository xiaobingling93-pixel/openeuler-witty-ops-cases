# 驱动升级或回退过程中出现版本校验不匹配

## 内核版本


## 问题现象
在Atlas 200T A2 Box16异构子框、Atlas 800T A2训练服务器、Atlas 800I A2推理服务器、Atlas 900 A2 PoD集群基础单元的驱动升级或回退过程中，出现以下报错之一：1) [ERROR]Required driver version crc error；2) [ERROR]Version check does not match, please use a newer version；3) [ERROR]Version check does not match, please use version xxx or later。同时提示驱动升级失败，详细信息记录在/var/log/ascend_seclog/ascend_install.log中。

## 问题根因
驱动版本与当前环境不适配或存在硬件故障。

## 解决方案
针对不同报错采取不同措施：报错1需联系华为技术支持处理；报错2应升级至较新版本驱动或联系技术支持确认兼容版本；报错3需按以下步骤操作：1) 安装或升级驱动至报错提示的版本；2) 重启Host系统；3) 执行npu-smi info命令检查是否丢卡，若未丢卡则继续升级配套固件并重启系统；若仍丢卡，则采用解决方案2：通过iBMC下电设备，使用带外方式升级与驱动配套的固件（具体操作参考对应设备的升级指导书），再上电并升级驱动至指定版本。若上述操作后仍显示丢卡，请联系华为技术支持。
