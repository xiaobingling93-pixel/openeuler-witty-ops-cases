# 兼容性测试工具报告显示异常的解决方法

## 内核版本


## 问题现象
兼容性测试工具报告显示异常，打开Chrome浏览器开发者工具，发现有“structuredClone is not defined”或“Uncaught SyntaxError: Unexpected token '.'”报错。

## 问题根因
兼容性测试报告部分代码使用了较新的JavaScript语法（如structuredClone和可选链操作符?.），在较低版本的浏览器上无法解析。

## 解决方案
1. 对于“structuredClone is not defined”错误：进入compatibility_testing.tar.gz解压后的文件夹，打开template.html.template文件，将第134行代码修改为 const dataCopy = JSON.parse(JSON.stringify(pipelineData[target]?.[${target}_tb_data]));
2. 对于“Uncaught SyntaxError: Unexpected token '.'”错误：在template.html.template文件中，将所有“?.[]”形式的“?.”去掉，将所有“?.name”形式的“?”去掉。
3. 修改完成后再次执行兼容性测试。
