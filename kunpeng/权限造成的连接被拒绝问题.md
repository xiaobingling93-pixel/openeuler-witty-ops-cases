# 权限造成的连接被拒绝问题

## 内核版本


## 问题现象
HTCondor服务因权限配置不足导致连接被拒绝。

## 问题根因
ADVERTISE_MASTER守护进程权限不足，condor_config文件中未正确配置ALLOW_ADVERTISE_MASTER参数，且在condor-8.9.*以上版本仅配置ALLOW_WRITE和ALLOW_READ不够，还需启用SECURITY: HOST_BASED策略。

## 解决方案
1. 在condor_config文件中添加ALLOW_ADVERTISE_MASTER = 192.168.47.42（支持子网如192.168.47.*）；
2. 若需详细日志，设置ALL_DEBUG = D_ALL；
3. 对于condor-8.9.*及以上版本，需添加use SECURITY : HOST_BASED；
4. 确保配置项等号两侧有空格，例如：ALLOW_WRITE  =  192.168.47.42, 192.168.47.111；
5. 根据具体报错（如ADVERTISE_SCHEDD权限不足），相应配置ALLOW_ADVERTISE_SCHEDD等参数。
