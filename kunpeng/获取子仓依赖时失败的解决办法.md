# 获取子仓依赖时失败的解决办法

## 内核版本


## 问题现象
在获取PyTorch子仓依赖时出现克隆失败错误，例如：'fatal: clone of 'https://github.com/NVlabs/cub.git' into submodule path '/home/pytorch/third_party/cub' failed' 或 'fatal: unable to access 'https://gitlab.com/libeigen/eigen.git/': error:1408F10B:SSL routines:ssl3_get_record:wrong version number'。

## 问题根因
网络不稳定导致子仓下载失败。

## 解决方案
方法一：进入PyTorch目录，执行 'git submodule update --init --recursive' 重新获取子仓。方法二：若方法一失败，可手动在可访问外网的机器上克隆子仓（如 eigen），上传至服务器指定路径，然后通过 'git config submodule.<module_name>.url <local_path>' 修改子仓URL为本地路径，再执行 'git submodule update --init --recursive' 完成初始化。
