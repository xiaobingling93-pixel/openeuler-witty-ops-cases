# 使用鲲鹏兼容性测试工具提示当前环境为非空问题的解决方法

## 内核版本


## 问题现象
在使用鲲鹏兼容性测试工具时，提示“当前环境为非空”，即系统中已存在数据或后台进程，可能影响测试结果的可靠性。

## 问题根因
系统后台存在占用CPU、内存、硬盘带宽或网络连接数较高的进程，例如ksoftirqd进程占用CPU资源100%，通常由网卡驱动问题引起。

## 解决方案
1. 查看工具日志（路径为“当前目录/log/info.log_yyyymmdd”）确认具体资源占用情况。
2. 若CPU或内存利用率过高，使用top命令定位并停止高占用进程。
3. 若硬盘带宽或网络连接数超标，使用iostat -d –x和netstat命令检查并停止相关进程。
4. 若发现ksoftirqd进程占用CPU 100%，需升级服务器网卡驱动：下载对应版本的1822网卡驱动（如NIC-IN200-CentOS7.6-hinic-xxxx-aarch64.rpm），通过rpm安装，然后卸载（rmmod hinic）并重新加载驱动（modprobe hinic），最后通过modinfo hinic |grep -i version验证版本。
