# 权限造成的连接被拒绝问题

## 内核版本


## 问题现象
由于权限配置不当，导致连接被拒绝。

## 问题根因
ADVERTISE_MASTER守护进程权限不足，condor_config文件中缺少正确的ALLOW_ADVERTISE_MASTER配置，或未启用HOST_BASED安全策略（condor-8.9.*以上版本）。此外，配置参数等号间缺少空格也会导致配置不生效。

## 解决方案
1. 在condor_config文件中正确配置允许的IP地址，例如：ALLOW_ADVERTISE_MASTER = 192.168.47.42（支持子网如192.168.47.*）。
2. 对于condor-8.9.*及以上版本，需添加：use SECURITY : HOST_BASED。
3. 确保配置项等号两侧有空格，例如：ALLOW_WRITE = 192.168.47.42, 192.168.47.111。
4. 如需详细日志排查，可设置：ALL_DEBUG = D_ALL。
