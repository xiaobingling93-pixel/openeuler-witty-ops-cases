# 某Java进程运行缓慢问题的解决方法

## 内核版本


## 问题现象
某Java进程出现运行缓慢，请求时延异常偏高，日志中出现OutOfMemoryError：Java heap space异常。

## 问题根因
代码中Map对象周期性地添加String数据，但未清除历史数据，导致堆内存持续增长，最终引发内存泄漏。

## 解决方案
通过jstat确认频繁Full GC且老年代内存使用率接近100%；增大堆内存后问题仍复现；使用jmap导出堆内存文件，通过jvisualvm分析发现大量String对象；定位到代码中Map未清理数据的问题；修改代码清除无用数据后重新编译部署，问题解决。
