# 生成的测试用例有权限问题的解决方法

## 内核版本


## 问题现象
生成的测试用例在运行时出现权限问题，具体表现为JVM沙箱环境限制了必要的运行权限，如退出虚拟机（exitVM.1）或对临时文件路径的写入权限，相关错误界面见图1和图2。

## 问题根因
JVM运行时所在的沙箱未配置相应的使用权限，导致测试用例执行过程中因缺少RuntimePermission或FilePermission而失败。

## 解决方案
在用户主目录下的.utgen/sandbox.policy文件中添加对应的Java沙箱安全策略：对于exitVM权限问题，添加'grant { permission java.lang.RuntimePermission "exitVM.1"; };'；对于文件写入权限问题，可针对特定路径授权（如'permission java.io.FilePermission "C:\Users\Username\AppData\Local\Temp\poifiles\...", "write";'）或开放全部文件写入权限（'permission java.io.FilePermission "<<ALL FILES>>", "write";'）。具体策略语法参考Policy File and Syntax及Full list of permissions文档。
