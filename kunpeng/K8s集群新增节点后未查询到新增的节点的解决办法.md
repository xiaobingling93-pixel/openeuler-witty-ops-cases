# K8s集群新增节点后未查询到新增的节点的解决办法

## 内核版本


## 问题现象
K8s从节点执行kubeadm加入主节点命令后提示成功加入集群，但在主节点没有查到该节点。

## 问题根因
主从节点的K8s组件的版本不一致导致。

## 解决方案
1. 在创建集群之前需要先保证主从节点的K8s组件版本一致。如果主从节点的K8s组件版本不一致，需要替换为一致的K8s组件版本。执行如下命令查询版本号：
   ```
   docker images | grep k8s.gcr.io
   ```
2. 在主从节点的K8s组件版本一致的情况下，重新部署K8s。详细信息请参见《Kubernetes 1.18.20 部署指南（CentOS&openEuler）》。
