# 移植x86 crc32汇编指令的方法

## 内核版本


## 问题现象
编译错误：unknown mnemonic 'crc32q' -- 'crc32q (x3),x2'或operand 1 should be an integer register -- 'crc32b [sp,11],x0'，或unrecognized command line option ‘-msse4.2’。

## 问题根因
x86使用的是crc32b、crc32w、crc32l、crc32q汇编指令完成CRC32C校验值计算功能，而ARM64平台使用crc32cb、crc32ch、crc32cw、crc32cx 4个汇编指令完成CRC32C校验值计算功能，两者指令不兼容。

## 解决方案
使用ARM64的crc32cb、crc32ch、crc32cw、crc32cx指令分别替换x86的crc32b、crc32w、crc32l、crc32q指令，并在编译时添加编译参数-march=armv8+crc。具体替换方式为：crc32cb用于8位数据替换crc32b，crc32ch用于16位数据替换crc32w，crc32cw用于32位数据替换crc32l，crc32cx用于64位数据替换crc32q。同时需调整内联汇编语法以适配ARM64寄存器约束和操作数格式。
