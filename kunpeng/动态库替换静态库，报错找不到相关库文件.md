# 动态库替换静态库，报错找不到相关库文件

## 内核版本


## 问题现象
执行编译Snappy-java时提示错误：make: *** [target/commons-crypto-1.0.0-Default/libcommons-crypto.so] Error 1。

## 问题根因


## 解决方案
解决方案一：进入“commons-crypto-CRYPTO-1.0.0”目录，在Makefile文件中添加G++的编译选项“-Wl,-Bdynamic”，确保使用动态链接方式。解决方案二：安装静态库，通过Yum源安装所需要的glibc、libstdc++-static，命令为：yum install libstdc++-static.aarch64 -y 和 yum install glibc* -y。
