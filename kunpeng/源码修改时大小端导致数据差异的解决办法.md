# 源码修改时大小端导致数据差异的解决办法

## 内核版本


## 问题现象
测试时发现测试数据出现偏差，整型数据正常值为31（00000000 00011111），但实际得到偏差值为7936（00001111 00000000）。

## 问题根因
代码原先运行在PowerPC架构（默认大端字节序）上，移植到当前平台后未处理大小端差异，导致整型数据字节序不匹配。

## 解决方案
1. 添加Short类型大小端转换宏：#define SWAP16(x) ((((x)& 0xFF00) >> 8) | (((x)& 0x00FF) << 8))；2. 对目标数组逐元素进行大小端转换：for(int i = 0; i < RESOLUTION_SIZE; i++){ short tmp = *(pIsmDppsDem50km + i); tmp = SWAP16(tmp); *(pIsmDppsDem50km + i) = tmp; }
