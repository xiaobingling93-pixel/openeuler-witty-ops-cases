# 标题
pkgserver “dnsmasq服务启动日志报错”

# 基本信息
- 测试用例: oe_test_service_dnsmasq
- 系统版本: openEuler-24.03-LTS-SP3(aarch64/x86_64)
- 是否问题: 否
- 关联issue: 无

# 问题现象
测试执行过程中，检查到服务在启动过程中存在报错日志，提示/etc/hosts文件不存在

# 错误日志
```
+ journalctl --since '2026-02-09 16:37:01' -u dnsmasq.service
+ grep -i 'fail\|error'
+ grep -v -i 'DEBUG\|INFO\|WARNING'
2月 09 16:37:01 localhost.localdomain dnsmasq[83950]: failed to load names from /etc/hosts: No such file or directory
2月 09 16:37:11 localhost.localdomain dnsmasq[83963]: failed to load names from /etc/hosts: No such file or directory
+ CHECK_RESULT 0 0 1 'There is an error message for the log of dnsmasq.service'
```

# 问题根因
1. 环境中的/etc/hosts被误删除

# 解决方案
1. 非问题，修改用例，在用例中环境中判断是否存在/etc/hosts文件，不存在则使用touch命令创建该文件
touch /etc/hosts
修复pr:https://atomgit.com/openeuler/mugen/pull/4964
