# 标题
kernel “ltp执行file_attr04失败”

# 基本信息
- 测试用例: file_attr04
- 系统版本: openEuler-24.03-LTS-SP3、openEuler-24.03-LTS-SP2、openEuler-24.03-LTS-SP1、openEuler-24.03-LTS(aarch64、x86_64)
- 是否问题：否
- 关联issue: https://atomgit.com/src-openeuler/kernel/issues/13468

# 问题现象
ltp测试执行过程中，报错：file_attr04.c:28: TFAIL: file_getattr( fd->fd, FILENAME, attr, FILE_ATTR_SIZE_LATEST, 0) expected ENOTDIR: EFAULT (14)

# 问题根因
1. 内核暂不支持file_setattr系统调用，原因为上游已支持这个特性，ltp20250930版本引入了用例，但OLK6.6还未引入这个特性
