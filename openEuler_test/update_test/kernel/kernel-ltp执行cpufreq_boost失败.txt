# 标题
kernel “ltp执行cpufreq_boost失败”

# 基本信息
- 测试用例: cpufreq_boost
- 系统版本: openEuler-24.03-LTS-SP3/openEuler-24.03-LTS-SP2/openEuler-24.03-LTS-SP1/openEuler-24.03-LTS/openEuler-24.03-LTS/openEuler-20.03-LTS-SP4/openEuler-22.03-LTS-SP4(aarch64、x86_64)
- 是否问题：否
- 关联issue: https://gitee.com/openeuler/kernel/issues/I493GF?from=project-issue&search_text=cpufreq_boost

# 问题现象
ltp测试执行过程中，报错：cpufreq_boost: TBROK: cpufreq_boost.c:175:Remaining cases broken

# 错误日志

# 问题根因
1. Kunpeng920 arm64服务器硬件不支持cpu超频（boost），BIOS setup下能耗相关选项设置efficient后，OS将加载cppc_cpufreq驱动，但操作/sys/devices/system/cpu/cpufreq/boost文件并不能使能boost,因为硬件并不支持。

# 解决方案
1. 可修改用例: 判断加载的驱动为cppc_cpufreq，返回不支持
static const struct cpufreq_driver_info cdrv[] = {
        { "acpi_cpufreq", 0, "1", "0", SYSFS_CPU_DIR "cpufreq/boost" },
        { "intel_pstate", 0, "1", "0", SYSFS_CPU_DIR "intel_pstate/no_turbo" },
        { "cppc_cpufreq", 0, "1", "0", SYSFS_CPU_DIR "cpufreq/boost" }

};
