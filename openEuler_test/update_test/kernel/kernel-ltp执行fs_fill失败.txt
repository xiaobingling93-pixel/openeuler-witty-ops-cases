# 标题
kernel “ltp执行fs_fill失败”

# 基本信息
- 测试用例: fs_fill
- 系统版本: openEuler-24.03-LTS-SP3、openEuler-24.03-LTS-SP2、openEuler-24.03-LTS-SP1、openEuler-24.03-LTS(aarch64)

# 问题根因
1. fs_fill 是一个压力测试，它通过多线程并发在文件系统中创建大量文件，用于测试文件系统的稳定性、并发能力和资源限制。测试已经进入清理阶段（正在执行 unlink 删除文件），但在清理过程中触发了超时，测试框架强制发送 SIGKILL 终止了测试

# 解决方案
1. 增加超时时间， export LTP_TIMEOUT_MUL=5
