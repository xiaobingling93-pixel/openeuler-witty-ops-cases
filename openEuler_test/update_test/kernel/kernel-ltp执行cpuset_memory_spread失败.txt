# 标题
kernel “ltp执行cpuset_memory_spread失败”

# 基本信息
- 测试用例: cpuset_memory_spread
- 系统版本: openEuler-24.03-LTS-SP3、openEuler-24.03-LTS-SP2、openEuler-24.03-LTS-SP1、openEuler-24.03-LTS、openEuler-22.03-LTS-SP4、openEuler-20.03-LTS-SP4(aarch64、x86)

# 问题根因
1. 概率性失败，测试用例设计不严谨：用例判断进程执行前后各个node上的page cache差值，PASS的范围是在2000-10000之间，但是 memory_spread 的配置只能设置分配的倾向性，不能保证具体在哪个node上分配多少，不能保证各个 node 之前page cache 的数量上的差值。


# 解决方案
1. 当前社区已经在讨论修改这个用例
2. 可单独执行此用例，大概率会成功，进入/opt/ltp，执行./runltp -s cpuset_memory_spread
