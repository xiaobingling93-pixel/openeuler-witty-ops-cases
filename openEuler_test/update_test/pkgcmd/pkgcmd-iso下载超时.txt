# 标题

# 基本信息
- 测试用例: oe_test_osinfo-detect
- 系统版本: openEuler-24.03-LTS-SP3(aarch64、x86_64)

# 问题根因
1、openEuler官方发布的iso镜像大小为3G，下载慢，mugen框架设置用例执行超时时间为30分钟，在30分钟内，iso镜像未下载成功

# 解决方案
1、非问题，修改测试用例，将测试的标准ISO镜像替换成大小为680M左右的netinst ISO镜像，如果仍然下载超时，建议检查网络，或者手动下载
修复pr：https://atomgit.com/openeuler/mugen/pull/4954
