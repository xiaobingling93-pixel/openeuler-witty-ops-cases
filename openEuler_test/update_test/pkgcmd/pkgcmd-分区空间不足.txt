# 标题
pkgcmd “分区空间不足”

# 基本信息
- 测试用例: oe_test_high_nbde
- 系统版本: openEuler-24.03-LTS-SP3(aarch64/x86_64)
- 是否问题: 否
- 关联issue: 无

# 问题现象
测试执行过程中，检测到测试环境获取的磁盘分区失败，提示空间不足

# 问题根因
1、测试用例获取空闲磁盘的方式不正确，导致获取到了系统盘，系统盘已无可用的分区空间

# 解决方案
1、非问题，修改测试用例，在测试用例中使用mugen测试框架的TEST_DISK方法获取空闲测试磁盘
修复PR：https://atomgit.com/openeuler/mugen/pull/4946
