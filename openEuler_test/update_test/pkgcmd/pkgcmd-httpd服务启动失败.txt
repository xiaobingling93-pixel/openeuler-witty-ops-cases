# 标题
pkgcmd “httpd服务启动失败”

# 基本信息
- 测试用例: oe_test_openssl_Implements_https
- 系统版本: openEuler-24.03-LTS-SP3
- 是否问题: 否
- 关联issue: 无

# 问题现象
systemctl restart httpd 执行失败，httpd服务启动报错。

# 问题根因
1、httpd试图监听的 80 端口被系统中的另一个进程占用了。
Apr 09 10:58:29 localhost.localdomain httpd[23269]: (98)Address already in use: AH00072: make_sock: could not bind to address [::]:80
Apr 09 10:58:29 localhost.localdomain httpd[23269]: (98)Address already in use: AH00072: make_sock: could not bind to address 0.0.0.0:80

# 解决方案
1、非问题，使用netstat -nltp | grep ":80" 查找占用80端口的进程，停止占用端口的进程。
2、重启httpd服务即可。
