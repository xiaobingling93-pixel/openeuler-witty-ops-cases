# 标题
pkgmanage “file XX from XX conflicts with file from package”

# 基本信息
- 测试用例: oe_test_pkg_manager01
- 系统版本: openEuler-24.03-LTS-SP3(aarch64)

# 问题现象
pkgmanage测试执行过程中，会存在安装完everything的包之后从而安装epol的包导致文件冲突。

# 错误日志
```
The downloaded packages were saved in cache until the next successful transaction.
You can remove cached packages by executing 'yum clean packages'.
Error: Transaction test error:
  file /usr/bin/typer from install of python3-typer-cli-0.13.1-1.oe2403sp3.noarch conflicts with file from package erlang-dialyzer-25.3.2.6-13.oe2403sp3.aarch64

```
# 问题根因
1、安装完epol软件包之后未卸载，从而导致和everything中某个软件包提供的文件一致，从而会安装冲突

# 解决方案
1. 因为属于不同的源，该问题是可以判定为误报，但是需要找环境单独分别安装两个软件包，排查是否可以正常安装
