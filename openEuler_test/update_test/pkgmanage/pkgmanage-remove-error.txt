# 标题
pkgmanage “Remove packages error !”

# 基本信息
- 测试用例: oe_test_pkg_manager01
- 系统版本: openEuler-24.03-LTS-SP3(aarch64)

# 问题现象
pkgmanage测试执行过程中，卸载软件包有报错信息，提示卸载失败。

# 错误日志
```
+ test -s /home/pkg_manager_folder/remove_fail_list
+ CHECK_RESULT 0 1 0 'Remove packages error !'
+ actual_result=0
+ expect_result=1
+ mode=0
+ error_log='Remove packages error !'

```

# 问题根因
1. 软件包卸载的过程中执行卸载脚本时候出现faild，error，warning信息，也会提示卸载失败
2、软件包卸载的时候，依赖属于受保护的包，所以会卸载失败

# 解决方案
1. 从 /home/pkg_manager_folder/remove_fail_list中确认是哪个软件包失败，在 /home/pkg_manager_folder/目录中找到对应的软件包查看日志，或者找干净环境复现。
2. 从案例库中搜索对应软件包的案例来分析日志
