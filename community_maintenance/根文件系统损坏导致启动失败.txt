#标题
根文件系统损坏导致启动失败

#内核版本
22.03 SP3

#问题现象
系统启动时无法挂载根文件系统，报错“Failed to start File System on /dev/mapper/openeuler-root”；日志显示根文件系统包含错误，需强制检查；systemd-fsck-root.service 启动失败，系统无法继续引导；复现方式为正常加电启动设备进入安装流程即可触发。

#问题根因
文件系统因异常关机或写入中断导致元数据不一致，journal 中残留未提交事务，而超级块的 recovery 标志未正确清除，触发 fsck 强制检查但无法自动修复。

#解决方案
1. 使用 Live CD 或救援模式启动系统；  
2. 执行命令 `fsck -y /dev/mapper/openeuler-root` 手动修复文件系统；  
3. 若提示超级块问题，可尝试使用备份超级块恢复：`e2fsck -b 32768 /dev/mapper/openeuler-root`（适用于 ext4 文件系统）。
