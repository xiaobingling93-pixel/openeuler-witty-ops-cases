#标题  
UnixBench 无法正确读取 CPU 核心数  

#内核版本  
24.03 SP1  

#问题现象  
在 openEuler 24.03 LTS SP1 系统上运行系统自带的 UnixBench 工具时，无法正确识别实际 CPU 核心数量，默认仅使用单核或固定核心数（如16）进行性能测试，导致并发线程数与实际 CPU 核心数不符，影响基准测试准确性。复现方式为直接执行预装的 UnixBench 测试命令即可观察到该异常行为。  

#问题根因  
旧版 UnixBench（低于6.0）在检测 CPU 核心数时依赖过时的系统接口或存在硬编码限制（如 maxCopies => 16），未能适配 openEuler 24.03 SP1 的内核或 /proc/cpuinfo 格式变化，导致 nproc 或相关检测逻辑失效。  

#解决方案  
1. 下载并解压 UnixBench 6.0 源码：tar -xf byte-unixbench-6.0.0.tar.gz  
2. 进入目录并编译：cd byte-unixbench-6.0.0/UnixBench && make all  
3. 动态替换最大副本数为实际 CPU 核心数：sed -i "s/\('system.*'maxCopies'\) => 16/\\1 => `nproc`/" Run
