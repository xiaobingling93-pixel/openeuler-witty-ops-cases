#标题
ROCE网卡驱动空指针解引用导致系统重启

#内核版本
2203sp4

#问题现象
- 业务运行过程中机器发生意外重启  
- 内核日志中反复出现 `[ROCE] roce3_event_report_common_b: [ofed event type] Wq access error. function id: 0`  
- 随后触发 `Unable to handle kernel NULL pointer dereference at virtual address 0000000000000000`  
- Oops 信息显示 PC 指向 `hmm_rdma_mtt_free+0x80/0x160 [hiroce3]`，调用栈涉及 `rpcrdma` 和 `ib_core` 模块  
- 系统最终进入 crashdump 并重启

#问题根因
是 rpcrdma 的已知问题触发的，不能使用原生 ko，dpc 安装包里的 rpcrdma.ko 是修复过这个问题的。根本原因在于使用了未修复的原生 rpcrdma.ko 模块，在处理 ROCE WQ 访问错误时，导致 hiroce3 驱动在 `hmm_rdma_mtt_free` 函数中解引用空指针，引发内核 Oops 并系统重启。

#解决方案
必须使用 DPC 安装包中提供的、已修复该问题的 rpcrdma.ko 模块，禁止使用操作系统原生或未经修复的 rpcrdma 内核模块。部署前应验证所加载的 rpcrdma 模块版本是否包含针对该空指针解引用问题的修复。
