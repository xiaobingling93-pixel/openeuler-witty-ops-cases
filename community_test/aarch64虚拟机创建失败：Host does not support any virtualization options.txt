#标题  
aarch64虚拟机创建失败：Host does not support any virtualization options

#内核版本  
2403sp1

#问题现象  
在使用 virt-install 创建 aarch64 架构虚拟机时，报错 “ERROR Host does not support any virtualization options”。日志显示 libvirt 报错 “internal error: Cannot find suitable emulator for aarch64”，且 getVersion 调用失败。尽管已安装依赖组件，仍无法正常启动虚拟机。

#问题根因  
系统缺少适用于 aarch64 架构的虚拟化模拟器（emulator），导致 libvirt 无法找到合适的 emulator 来运行虚拟机，从而触发 “Cannot find suitable emulator for aarch64” 错误。

#解决方案  
1. 确认已安装 qemu-system-aarch64 或 qemu-kvm 包（针对 aarch64 架构的 QEMU 模拟器）。
2. 检查 /usr/libexec/qemu-kvm 或 /usr/bin/qemu-system-aarch64 是否存在并可执行。
3. 若未安装，执行以下命令安装所需组件（以 openEuler 为例）：
   ```bash
   dnf install qemu-system-aarch64
   ```
4. 验证 libvirt 是否能正确识别 aarch64 emulator：
   ```bash
   virsh capabilities | grep -A5 "arch name='aarch64'"
   ```
5. 确保 BIOS/固件中已启用 ARM 虚拟化支持（如适用）。
