#标题  
axel软件包缺失导致测试套执行失败  

#内核版本  
2403sp3  

#问题现象  
- 在ARM架构环境执行测试套`oe_test_axel_01`时，运行`yum list axel`命令返回错误：`Error: No matching Packages to list`  
- 脚本输出明确提示：`axel does not exist`  
- 测试流程以退出码`255`异常终止  
- 日志时间戳：2026-01-13 11:03:45 CST，元数据缓存检查间隔正常（0:00:18）  

#问题根因  
axel软件包在当前系统配置的软件源中不存在（文档明确结论："axel软件包不存在"），导致依赖该包的测试用例无法通过环境校验。  

#解决方案  
1. **验证软件源配置**：检查`/etc/yum.repos.d/`下源配置，确认已启用包含axel的软件源（特别关注ARM架构适配源）  
2. **更新缓存重试**：执行`yum clean all && yum makecache`后重新查询  
3. **确认包名与架构支持**：  
   - 核查axel在目标OS版本（24.03-LTS-SP3）ARM仓库中是否提供（可尝试`yum search axel`）  
   - 排查包名拼写（如是否应为`axel2`等变体）  
4. **补充安装方案**：  
   - 若官方源无此包，添加可信第三方源或通过源码编译安装  
   - 测试环境预置阶段显式安装axel：`yum install -y axel`  
5. **测试脚本增强**：在测试套前置检查中增加软件源可用性验证逻辑，避免因环境缺失导致误判
