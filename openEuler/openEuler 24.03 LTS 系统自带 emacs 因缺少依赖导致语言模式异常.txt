#标题
openEuler 24.03 LTS 系统自带 emacs 因缺少依赖导致语言模式异常

#内核版本
24.03 LTS

#问题现象
安装系统默认源的 emacs 后，在使用 c-mode、python-mode 等语言模式时频繁报错：
error in process filter: Wrong type argument: stringp, (require . china-util)
error in process filter: Wrong type argument: stringp, nil
导致相关语言模式无法正常使用；此外，c-mode 中部分符号无法输入。

#问题根因
系统打包的 emacs 缺少必要的构建依赖（如 texinfo、json-c-devel 等），导致运行时功能缺失和类型错误。

#解决方案
1. 卸载系统自带 emacs，从源码重新编译安装。
2. 编译前安装必要依赖包：
   dnf install ncurses-devel libXt-devel libXpm-devel libXaw-devel gnutls-devel tcl-devel giflib-devel texinfo json-c-devel
3. 使用以下配置和编译命令：
   ./configure --without-pop
   make bootstrap
