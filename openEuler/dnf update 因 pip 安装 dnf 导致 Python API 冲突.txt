#标题
dnf update 因 pip 安装 dnf 导致 Python API 冲突

#内核版本
未知

#问题现象
执行 dnf update 或 yum install 时出现 ModuleNotFoundError: No module named 'dnf.cli'；同时提示 "/usr/local/lib/python3.9/site-packages/dnf.py:15: UserWarning: The DNF Python API is not currently available via PyPI"；系统中存在 /usr/lib/python3.9/site-packages/dnf 与 /usr/local/lib/python3.9/site-packages/dnf 两个路径。

#问题根因
用户通过 pip install dnf 安装了 PyPI 版本的 dnf，与系统包管理器提供的 dnf 包发生冲突，导致 Python 优先加载了非系统兼容的 dnf 模块。

#解决方案
1. 执行 pip uninstall dnf 卸载通过 pip 安装的 dnf 包。
