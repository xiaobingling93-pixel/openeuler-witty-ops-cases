#标题
升级过程中 java-11-openjdk-headless 安装失败导致事务错误

#内核版本
未知

#问题现象
从 CentOS 7.9 升级到 openEuler 22.03 时，升级过程报错 “Error: Transaction failed”，具体涉及 java-11-openjdk-headless-1 包；升级完成后系统自动安装了 openjdk-8-headless；重新安装 java-11-openjdk-headless-1 时提示目录已存在而失败。

#问题根因
旧版本 java-11-openjdk-headless-1 未完全清理，残留的 JVM 目录导致新版本安装冲突。

#解决方案
1. 手动卸载所有已安装的 openjdk 包；
2. 删除残留的 /usr/lib/jvm 目录（或相关 JVM 目录）；
3. 重新安装 java-11-openjdk-headless。
