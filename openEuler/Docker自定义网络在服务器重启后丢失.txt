#标题
Docker自定义网络在服务器重启后丢失

#内核版本
未知

#问题现象
在OpenEuler系统中使用 docker network create 创建的自定义网络，在服务器重启后消失，执行 docker network ls 无法看到该网络。

#问题根因
系统掉电或重启时，Docker daemon 默认会清理 data-root 下的 db 文件，导致重启前创建的网络等资源被删除。

#解决方案
1. 编辑 Docker 配置文件 /etc/sysconfig/docker。
2. 在文件最底部添加配置项：DISABLE_CRASH_FILES_DELETE=true。
