#标题
Docker 启动 MySQL 5.6/5.7 报错 mysqld failed while attempting to check config

#内核版本
未知

#问题现象
使用 Docker 运行 MySQL 5.6 或 5.7 官方镜像时无法启动，报错信息为 “mysqld failed while attempting to check config”；相同命令启动 MySQL 8.0 镜像则正常。

#问题根因
Docker 服务的 LimitNOFILE 设置为 infinity 导致 MySQL 5.6/5.7 启动时配置检查失败。

#解决方案
1. 编辑 Docker 服务配置文件（如 /usr/lib/systemd/system/docker.service）。
2. 将以下三行：
   LimitCORE=infinity
   LimitNOFILE=infinity
   LimitNPROC=infinity
   修改为：
   LimitCORE=65535
   LimitNOFILE=65535
   LimitNPROC=65535
3. 执行 systemctl daemon-reload 重载配置。
4. 执行 systemctl restart docker 重启 Docker 服务。
