#标题
yum install --allow-erasing 导致 librpmio.so.9 缺失 ZSTD_compressStream2 符号

#内核版本
4.19.90-2003.4.0.0036.oe1.x86_64

#问题现象
执行 yum install ncurses-devel --allow-erasing 后，yum 和 rpm 命令均无法使用，报错：ImportError: /usr/lib64/librpmio.so.9: undefined symbol: ZSTD_compressStream2（Python traceback）或 rpm: symbol lookup error: /usr/lib64/librpmio.so.9: undefined symbol: ZSTD_compressStream2。

#问题根因
--allow-erasing 参数导致系统卸载了与 ZSTD 相关的旧库，造成 librpmio.so.9 依赖的 ZSTD_compressStream2 符号缺失。

#解决方案
1. 挂载系统安装镜像进入救援模式；
2. 创建临时挂载点并挂载光盘设备（如 /dev/sr0）；
3. 进入镜像 Packages 目录，使用 rpm -ivh --root=/mnt/sysimage/ --force --nodeps 强制重装被卸载的相关 RPM 包。
