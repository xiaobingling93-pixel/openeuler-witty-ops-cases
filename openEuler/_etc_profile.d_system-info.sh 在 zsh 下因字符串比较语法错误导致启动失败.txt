#标题
/etc/profile.d/system-info.sh 在 zsh 下因字符串比较语法错误导致启动失败

#内核版本
未知

#问题现象
在使用 zsh 作为默认 shell 时，登录或执行 shell 初始化脚本过程中报错：zsh: = not found；问题源于 /etc/profile.d/system-info.sh 第81-86行使用了双等号（==）进行字符串比较。

#问题根因
/etc/profile.d/system-info.sh 脚本中使用了 bash 特有的双等号（==）字符串比较操作符，而 zsh 不兼容该语法，导致解析失败。

#解决方案
将脚本中的字符串比较操作符从 == 改为 POSIX 兼容的单等号（=），具体修改如下：
if [ "$(whoami)" = "root" ]
then
    echo -e "\n"
else
    echo -e "To run a command as administrator(user \"root\"),use \"sudo\n\"."
fi
