#标题
Zabbix Server 启动失败因 net-snmp 版本不兼容

#内核版本
24.03 LTS SP3

#问题现象
执行 systemctl restart zabbix-server 后服务启动失败，日志显示：  
/usr/sbin/zabbix_server: symbol lookup error: /usr/sbin/zabbix_server: undefined symbol: usmAES256CiscoPrivProtocol  
systemd 报错：Control process exited, code=exited, status=127/n/a，最终状态为 failed。

#问题根因
系统自带的 net-snmp-libs 与 Zabbix 所需的 net-snmp 版本不一致，新版本符号表变更导致 Zabbix Server 运行时无法找到所需符号 usmAES256CiscoPrivProtocol。

#解决方案
1. 卸载系统自带的 net-snmp-libs 和 net-snmp：  
   yum remove net-snmp-libs net-snmp  
2. 添加 openEuler-24.03-LTS-SP1 的 extras 源（oepkgs.repo），内容如下：  
   [oepkgs]  
   name=oepkgs openEuler Extras  
   baseurl=https://repo.oepkgs.org/rpm/openEuler-24.03-LTS-SP1/extras/x86_64/  
   enabled=1  
   gpgcheck=0  
   priority=95  
3. 安装兼容版本的 net-snmp-libs 和 net-snmp：  
   dnf install net-snmp-libs net-snmp  
4. 若已安装 Zabbix，先卸载所有相关包（包括 zabbix-release.noarch）：  
   yum remove zabbix-server-pgsql zabbix-web-pgsql zabbix-apache-conf zabbix-sql-scripts zabbix-selinux-policy zabbix-agent zabbix-release.noarch  
5. 安装 Zabbix 7.0 LTS for CentOS 9：  
   rpm -Uvh https://repo.zabbix.com/zabbix/7.0/centos/9/x86_64/zabbix-release-latest-7.0.el9.noarch.rpm  
   dnf clean all  
   在 Zabbix 源配置中添加 priority=95  
   dnf install zabbix-server-mysql zabbix-web-mysql zabbix-apache-conf zabbix-sql-scripts zabbix-selinux-policy zabbix-agent  
6. 启动服务：systemctl restart zabbix-server  
7. 安装额外依赖：php-fpm、php-pgsql、zabbix-sender、zabbix-get  
8. 配置 openGauss 使用 MD5 认证：  
   - 修改 /var/lib/opengauss/data/pg_hba.conf，设置认证方式为 MD5 或 scram-sha-256  
   - 执行 gsql 命令：ALTER SYSTEM SET password_encryption_type = 1;（或 =3）  
   - 执行 SELECT pg_reload_conf();  
   - 重置数据库用户密码
