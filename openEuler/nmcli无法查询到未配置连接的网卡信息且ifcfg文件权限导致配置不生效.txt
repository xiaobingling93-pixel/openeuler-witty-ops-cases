#标题
nmcli无法查询到未配置连接的网卡信息且ifcfg文件权限导致配置不生效

#内核版本
未知

#问题现象
1. 使用 nmcli c 命令无法查看到 eno1 网卡信息，但 ip a 或 ifconfig 可正常显示该网卡。
2. nmcli d 显示 eno1 状态为 disconnected。
3. 配置文件 /etc/sysconfig/network-scripts/ifcfg-eno1 中的地址不生效。
4. journalctl -u NetworkManager 日志中出现 “Could not read file ‘/etc/sysconfig/network-scripts/ifcfg-eno1’: Permission denied” 报错。

#问题根因
nmcli c 仅显示已创建的连接（connection），未为 eno1 创建连接前无法显示；此外，NetworkManager 因无权限读取 ifcfg-eno1 文件导致配置不生效。

#解决方案
1. 为网卡手动创建连接：nmcli con add type ethernet con-name eno1 ifname eno1 ip4 192.168.0.10/24。
2. 激活连接：nmcli con up eno1。
3. 排查并修复 /etc/sysconfig/network-scripts/ifcfg-eno1 文件的权限，确保 NetworkManager 有读取权限。
