#标题
simpleVulkan运行时因缺少libglfw.so.3及未设置XDG_RUNTIME_DIR导致启动失败

#内核版本
未知

#问题现象
执行simpleVulkan二进制程序时报错：  
1. “error while loading shared libraries: libglfw.so.3: cannot open shared object file: No such file or directory”；  
2. 设置LD_LIBRARY_PATH后再次运行，提示“error: XDG_RUNTIME_DIR is invalid or not set in the environment”，但程序仍继续输出FPS日志。

#问题根因
程序运行依赖libglfw.so.3动态库未在链接器搜索路径中，且环境变量XDG_RUNTIME_DIR未正确设置。

#解决方案
1. 将libglfw.so.3所在目录（如/usr/local/lib64）加入LD_LIBRARY_PATH环境变量；  
2. 设置XDG_RUNTIME_DIR环境变量指向有效的运行时目录（如/tmp/runtime-root）。
