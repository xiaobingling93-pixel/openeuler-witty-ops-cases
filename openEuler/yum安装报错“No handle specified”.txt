#标题
yum安装报错“No handle specified”

#内核版本
未知

#问题现象
执行 yum -y install telnet 命令时，报错“Error: No handle specified”；日志显示“Unable to connect to Registration Management Service”，并抛出 RuntimeError: No handle specified 及 dnf.exceptions.RepoError: No handle specified。

#问题根因
YUM 仓库配置中缺少有效的句柄（handle），导致 libdnf 在加载仓库时无法识别源。

#解决方案
1. 执行 `yum clean all` 清理 YUM 缓存。
2. 执行 `yum makecache` 重新生成元数据缓存。
