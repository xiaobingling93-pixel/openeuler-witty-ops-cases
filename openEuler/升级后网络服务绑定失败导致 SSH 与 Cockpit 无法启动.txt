#标题
升级后网络服务绑定失败导致 SSH 与 Cockpit 无法启动

#内核版本
6.6.0-93.0.0.98.oe2403sp1

#问题现象
系统升级后，sshd 和 cockpit.socket 服务启动失败，日志中反复出现“绑定到端口 XXXX 上的 IP 地址失败：无法分配请求的地址”错误，具体包括端口 39888、39 和 1888 的绑定失败，导致 SSH 和 Cockpit 服务无法自动启动。

#问题根因
系统升级后，指定 IP 地址在服务启动时不可用或未正确配置，导致监听套接字无法绑定到该地址。

#解决方案
1. 检查并确保服务配置中指定的 IP 地址在系统启动时已正确配置并处于可用状态。
2. 若服务无需绑定特定 IP，可修改 sshd 和 cockpit 配置，将监听地址改为 0.0.0.0 或 :: 以监听所有接口。
