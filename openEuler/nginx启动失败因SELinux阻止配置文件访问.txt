#标题
nginx启动失败因SELinux阻止配置文件访问

#内核版本
未知

#问题现象
执行 systemctl start nginx.service 失败，日志显示 "nginx: [emerg] open() “/etc/nginx/default.d/uwsgi.conf” failed (13: Permission denied)"，配置测试失败；关闭 firewalld、清空 iptables 规则并执行 setenforce 0 后可成功启动。

#问题根因
SELinux 处于 enforcing 模式，阻止 nginx 进程访问 /etc/nginx/default.d/uwsgi.conf 文件。

#解决方案
1. 执行 systemctl stop firewalld  
2. 执行 iptables -F  
3. 执行 setenforce 0 以临时禁用 SELinux
