#标题
22.03 LTS 中 SSH 无法通过 hosts.allow 实现登录白名单限制

#内核版本
未知

#问题现象
配置 /etc/hosts.deny 为 "sshd:ALL"、/etc/hosts.allow 为 "sshd:192.168.1.1" 后，预期仅允许 192.168.1.1 登录，但其他 IP 仍可正常 SSH 登录服务器。

#问题根因
OpenSSH 自 6.6 版本起移除了对 tcp_wrappers 的支持，导致 sshd 无法通过 /etc/hosts.allow 和 /etc/hosts.deny 实现访问控制。

#解决方案
1. 使用 iptables 对 SSH 登录进行 IP 访问限制。
2. 或通过修改 /etc/security/access.conf 文件来禁止或允许特定 IP 或 IP 段访问。
