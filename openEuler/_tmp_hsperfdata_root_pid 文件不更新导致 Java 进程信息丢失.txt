#标题
/tmp/hsperfdata_root/pid 文件不更新导致 Java 进程信息丢失

#内核版本
未知

#问题现象
在 openEuler 22.03 LTS 系统下，Java 进程在 /tmp/hsperfdata_root/pid 的进程信息文件长时间不更新，始终保留进程启动时的时间戳；系统因该文件长时间未更新而将其清理，导致无法使用 JDK 1.8 自带的 jstack 命令获取堆栈信息。

#问题根因
/tmp 目录挂载为 tmpfs 文件系统导致 hsperfdata 文件无法正常更新。

#解决方案
1. systemctl mask tmp.mount  
2. umount -l /tmp 卸载 /tmp 目录  
3. 重启应用
