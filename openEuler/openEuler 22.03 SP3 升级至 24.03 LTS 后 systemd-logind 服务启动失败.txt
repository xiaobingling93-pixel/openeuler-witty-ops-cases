#标题
openEuler 22.03 SP3 升级至 24.03 LTS 后 systemd-logind 服务启动失败

#内核版本
未知

#问题现象
从 openEuler 22.03 LTS SP3 升级到 24.03 LTS 后，系统重启时卡在 “FAILED failed to start user login management” 错误界面，无法正常进入系统。升级过程中出现警告：“文件 /usr/lib64/samba/wbclient/libwbclient.so.0.15 删除失败”，升级后该文件及软链接 /usr/lib64/samba/wbclient/libwbclient.so.0 均不存在。

#问题根因
升级过程中 samba 相关库文件被错误处理导致 systemd-logind 依赖缺失，进而使 systemd-logind.service 启动失败。

#解决方案
1. 若尚未重启：执行 `dnf reinstall systemd` 后再重启。
2. 若已重启且无法进入系统：
   - 使用 Live CD 启动并 chroot 到原系统；
   - 执行以下命令修复关键组件：
     ```
     dnf reinstall systemd systemd-udev
     mv /etc/selinux/targeted /etc/selinux/targeted.bak
     mv /etc/selinux/config /etc/selinux/config.bakold
     dnf remove selinux-policy*
     dnf install selinux-policy-targeted
     dnf install selinux-policy-devel policycoreutils policycoreutils-devel
     mv /etc/selinux/config /etc/selinux/config.bakrpmnew
     mv /etc/selinux/config.bakold /etc/selinux/config
     touch /.autorelabel
     ```
   - 重启系统，等待 SELinux 自动重建标签并再次重启后恢复正常。
