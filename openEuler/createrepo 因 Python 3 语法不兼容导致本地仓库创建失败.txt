#标题
createrepo 因 Python 3 语法不兼容导致本地仓库创建失败

#内核版本
未知

#问题现象
执行 create local repo 操作时，报出 Python 脚本语法错误，具体为文件 /usr/share/createrepo/genpkgmetadata.py 第 207 行出现 "SyntaxError: invalid syntax"，错误指向 "except EnvironmentError, e:" 语句；日志显示 "[ ERROR ] - [centos2openEuler]: create local repo failed" 和 "[ ERROR ] - [centos2openEuler]: config local repo failed"。

#问题根因
createrepo 工具使用的脚本 /usr/share/createrepo/genpkgmetadata.py 采用 Python 2 风格的异常处理语法（except Exception, var），该语法在 Python 3 中已被废弃，导致在默认 Python 版本为 3.9 的环境中运行时报语法错误。

#解决方案
将系统默认 Python 版本切换为 Python 2.7 以兼容 createrepo 脚本。
