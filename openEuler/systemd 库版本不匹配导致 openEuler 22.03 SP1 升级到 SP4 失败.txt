#标题
systemd 库版本不匹配导致 openEuler 22.03 SP1 升级到 SP4 失败

#内核版本
22.03sp4

#问题现象
升级过程中 systemd-249-81.oe2203sp4.x86_64 安装失败，系统重启后无法进入；执行 udevadm 命令时报错：symbol lookup error: udevadm: undefined symbol: strv_copy_n, version SD_SHARED。

#问题根因
udevadm 与 libsystemd-shared.so 版本不一致，导致缺少 systemd 249+ 引入的 strv_copy_n 符号。

#解决方案
1. 进入救援模式：在 GRUB 启动菜单编辑启动参数，添加 systemd.unit=rescue.target，或使用 openEuler 22.03 SP4 安装 ISO 启动到 Rescue Mode。
2. 修复 systemd 依赖关系：
   - 执行 dnf reinstall systemd systemd-libs systemd-udev --allowerasing -y；
   - 若 dnf 不可用，使用 rpm -Uvh --force --nodeps 强制覆盖安装 systemd 相关 RPM 包。
3. 验证修复：
   - 使用 nm -D 检查 libsystemd-shared-*.so 是否包含 strv_copy_n 符号；
   - 测试 udevadm --version 和 udevadm info --root 是否正常；
   - 重建 initramfs：dracut -f --regenerate-all；
   - 更新 grub 配置：grub2-mkconfig -o /boot/grub2/grub.cfg。
4. 退出救援模式并重启系统。
