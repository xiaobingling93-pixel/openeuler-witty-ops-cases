#标题
xfs_vn_get_link 未检查 done 指针为空导致系统崩溃

#内核版本
22.03-LTS-SP4

#问题现象
在 openEuler-22.03-LTS-SP4 系统中，当 xfs_vn_get_link 函数调用 set_delayed_call(done, kfree_link, link) 时，若 done 指针为 NULL，会导致系统崩溃。

#问题根因
xfs_vn_get_link 函数未对 done 参数进行 NULL 检查，直接将其传入 set_delayed_call，当 done 为 NULL 时引发空指针解引用导致系统崩溃。

#解决方案
在 xfs_vn_get_link 函数中增加对 done 指针是否为 NULL 的检查，避免在 done 为空时调用 set_delayed_call。
