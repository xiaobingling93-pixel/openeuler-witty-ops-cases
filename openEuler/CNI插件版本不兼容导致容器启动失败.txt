#标题
CNI插件版本不兼容导致容器启动失败

#内核版本
未知

#问题现象
在openEuler2203系统安装nerdctl-1.7.6、cni-plugins-v1.5.0和containerd-v1.6.32后，首次启动容器报错“failed to allocate for range 0: 10.4.1.7 has been allocated”，第二次启动报错“failed to set bridge addr: could not set bridge’s mac: invalid argument”，第三次启动才正常。

#问题根因
cni-plugins-v1.5.0与openEuler2203系统存在兼容性问题，导致CNI bridge插件在容器启动时无法正确分配IP或设置MAC地址。

#解决方案
1. 下载并安装兼容的cni-plugins-v1.5.1版本：
   wget https://github.com/containernetworking/plugins/releases/download/v1.5.1/cni-plugins-linux-amd64-v1.5.1.tgz
2. 清空原有CNI插件目录：
   sudo rm -rf /opt/cni/bin/*
3. 解压新版本插件到指定目录：
   sudo tar -C /opt/cni/bin -xzvf cni-plugins-linux-amd64-v1.5.1.tgz
