#标题
容器内执行test -x / 返回值异常

#内核版本
未知

#问题现象
在 CentOS 7.9 物理机上运行 openEuler 22.03 容器时，执行 `test -x /` 后 `echo $?` 返回 1；而在 openEuler 22.03 宿主机上运行相同容器时返回 0。

#问题根因
Docker 19.03 的 seccomp 配置中缺少 faccessat2 系统调用白名单，导致 seccomp 直接返回错误给 glibc；低版本 runc（早于 1.0.0-rc93）未实现对新增系统调用号的兼容处理，无法将 faccessat2 调用回退至 faccessat，从而导致 glibc 报错。

#解决方案
1. 在 docker profiles/seccomp/seccomp_default.go 代码中添加 faccessat2 系统调用白名单  
2. 升级 containerd 版本至 1.6.8，以使用高版本 runc 的新增系统调用兼容功能  
3. 使用 openEuler 社区 22.03 版本的 docker
