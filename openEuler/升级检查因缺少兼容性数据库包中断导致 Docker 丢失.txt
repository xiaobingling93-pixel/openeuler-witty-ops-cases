#标题
升级检查因缺少兼容性数据库包中断导致 Docker 丢失

#内核版本
未知

#问题现象
在使用 x2openEuler 工具执行升级前检查时，任务中断并报错：“在节点上部署 [x2openEuler-database-centos7.5] 失败，上传安装包异常”。用户填写了 docker-ce 后触发该中断；若不填写，则升级检查通过但升级后 Docker 被移除。部分升级完成后系统进入救援模式。

#问题根因
升级工具在检测到用户指定 docker-ce 时需加载对应源操作系统与目标操作系统的兼容性数据库包（x2openEuler-database-*.rpm），若未提供匹配的 database 包，则兼容性检查失败导致升级流程中断；若跳过兼容性检查，工具无法保留不兼容的 Docker 组件，导致其在升级后被清理。

#解决方案
1. 根据源操作系统和目标操作系统版本，从官方路径下载对应的三个 database 包（源、目标、源-目标组合）。
2. 将下载的 RPM 包放入服务端 /etc/x2openEuler/rpms/databases 目录。
3. 设置文件权限为 440，属主为 x2openEuler:x2openEuler。
4. 若目标版本在工具中未直接列出，可手动填写符合格式的目标版本以匹配可用的 database 包。
