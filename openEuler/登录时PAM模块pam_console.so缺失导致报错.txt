#标题
登录时PAM模块pam_console.so缺失导致报错

#内核版本
未知

#问题现象
系统升级后登录时出现错误：PAM unable to dlopen(/usr/lib64/security/pam_console.so): /usr/lib64/security/pam_console.so: cannot open shared object file: No such file or directory

#问题根因
/etc/pam.d/login 配置文件中引用了已不存在的 pam_console.so 模块

#解决方案
注释 /etc/pam.d/login 配置文件中的以下行：
#session    optional     pam_console.so
